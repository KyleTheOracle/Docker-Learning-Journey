Module 1: Docker Basics

Lesson 1: What are Containers?

Containers are lightweight, standalone, executable software packages that include everything needed to run a piece of software: code, runtime, system tools, libraries, and settings.

Container vs VM

Unlike virtual machines, containers share the host system's kernel but provide isolated processes and filesystems.

  • Containers: Share OS kernel, lightweight, fast startup
  • VMs: Include entire OS, more isolated, resource-intensive

Lesson 2: Docker Architecture

Docker uses a client-server architecture with several key components working together.

Docker Components

  • Docker Client: Command-line interface
  • Docker Daemon: Background service that manages containers
  • Docker Registry: Stores Docker images (e.g., Docker Hub)
  • Docker Objects: Images, containers, networks, volumes

Get Docker up and running on your system with these simple steps.

Installation Options

  • Docker Desktop: For Windows and Mac

Lesson 2: Managing Containers

Learn how to work with running containers and control their lifecycle.

Container Lifecycle

# Run container in background
docker run -d nginx

# Run with a name
docker run --name my-nginx nginx

# Run with port mapping
docker run -p 8080:80 nginx

# Execute command in container
docker exec -it container_id bash

# View container logs
docker logs container_id

Common Docker Run Flags

  • -d: Run in detached mode (background)
  • -p: Map host port to container port
  • -v: Mount a volume
  • -e: Set environment variables
  • --name: Assign a name to the container
  • --network: Connect to a network

Lesson 3: Working with Images

Learn how to find, pull, and manage Docker images.

Image Commands

# Pull an image
docker pull ubuntu:20.04

# Search for images
docker search mysql

# Remove an image
docker rmi image_id

# Build an image from Dockerfile
docker build -t my-app:1.0 .

Image Tags and Versions

Docker images use tags to identify versions:

  • nginx:latest - Latest version
  • ubuntu:20.04 - Specific version
  • mysql:8.0 - Major version

Always specify a version tag for production use to ensure consistency.

Module 3: Dockerfiles

Lesson 1: Dockerfile Basics

Learn how to create Dockerfiles to build custom images for your applications.

Dockerfile Structure

# Simple Dockerfile example
FROM node:14
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["npm", "start"]

Common Dockerfile Instructions

  • FROM: Base image to build from
  • WORKDIR: Set working directory
  • COPY/ADD: Copy files into image
  • RUN: Execute commands during build
  • EXPOSE: Document container ports
  • ENV: Set environment variables
  • CMD/ENTRYPOINT: Default command to run

Lesson 2: Building Images

Learn how to build Docker images from your Dockerfiles and understand the build process.

Building Docker Images

Step-by-step guide to building custom images

Build Commands

# Basic build
docker build -t my-app:1.0 .

# Build with build args
docker build --build-arg ENV=prod -t my-app:prod .

# Build from specific Dockerfile

Lesson 3: Multi-stage Builds

Learn how to create efficient, smaller Docker images using multi-stage builds.

Multi-stage Build Example

# Build stage
FROM node:14 AS build
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# Production stage
FROM nginx:alpine
COPY --from=build /app/build /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

Benefits of Multi-stage Builds

  • Smaller final image size
  • Separation of build and runtime environments
  • No build tools in production image
  • Improved security with smaller attack surface
  • Faster deployments with smaller images

Module 4: Docker Networking

Lesson 1: Network Basics

Understand Docker's networking model and how containers communicate.

Docker Network Types

  • bridge: Default network for containers
  • host: Container shares host's network
  • none: No networking
  • overlay: Multi-host networking
  • macvlan: Assign MAC address to container
  • custom: User-defined networks

Docker Networking Explained

Visual guide to container networking

Lesson 2: Managing Networks

Learn how to create, inspect, and manage Docker networks.

Network Commands

# List networks
docker network ls

# Create a network
docker network create my-network

# Inspect a network
docker network inspect my-network

# Connect container to network
docker network connect my-network container_id

# Disconnect container
docker network disconnect my-network container_id

# Remove a network
docker network rm my-network

Container Communication

Containers on the same network can communicate using:

  • Container names as hostnames
  • Internal IP addresses
  • Exposed ports
# Example: Run two containers on same network
docker network create app-net
docker run -d --name db --network app-net mysql
docker run -d --name web --network app-net nginx

Lesson 3: Port Mapping

Learn how to expose container ports to the host system and the outside world.

Port Mapping Basics

# Map container port 80 to host port 8080
docker run -p 8080:80 nginx

# Map multiple ports
docker run -p 8080:80 -p 443:443 nginx

# Map to random host port
docker run -P nginx

# Map to specific interface
docker run -p 127.0.0.1:8080:80 nginx

Accessing Mapped Ports

After mapping ports, you can access the container:

  • From host: http://localhost:8080
  • From network: http://host-ip:8080

Check mapped ports with:

docker port container_id

Module 5: Docker Compose

Lesson 1: Compose Basics

Learn how to define and run multi-container applications with Docker Compose.

What is Docker Compose?

Docker Compose is a tool for defining and running multi-container Docker applications. With Compose, you use a YAML file to configure your application's services, networks, and volumes.

Benefits include:

  • Single command to start all services
  • Environment consistency
  • Simplified configuration
  • Automatic networking between services

Docker Compose Introduction

Visual guide to Docker Compose basics

Lesson 2: docker-compose.yml

Learn how to create a docker-compose.yml file to define your application stack.

Sample docker-compose.yml

version: '3'

services:
  web:
    build: ./web
    ports:
      - "8080:80"
    depends_on:
      - db
    environment:
      - DATABASE_URL=postgres://postgres:password@db:5432/app
      
  db:
    image: postgres:13
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_PASSWORD=password
      - POSTGRES_DB=app

volumes:
  postgres_data:

Key Compose Elements

  • version: Compose file format version
  • services: Container definitions
  • volumes: Persistent data storage
  • networks: Custom networks
  • depends_on: Service dependencies
  • environment: Environment variables
  • ports: Port mappings

Lesson 3: Compose Commands

Learn essential Docker Compose commands to manage your application stack.

Basic Compose Commands

# Start services
docker-compose up

# Start in detached mode
docker-compose up -d

# Stop services
docker-compose down

# Stop and remove volumes
docker-compose down -v

# View logs
docker-compose logs

# Build or rebuild services
docker-compose build

# List containers
docker-compose ps

Advanced Compose Usage

# Scale specific services
docker-compose up -d --scale web=3

# Run a command in a service
docker-compose exec web npm test

# View resource usage
docker-compose top

# Use a different compose file
docker-compose -f docker-compose.prod.yml up

# Restart a single service
docker-compose restart web
